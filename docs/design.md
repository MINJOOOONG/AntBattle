# Design

개미배틀 앱의 디자인 가이드 문서.

## 톤앤매너

- 귀엽고 웃긴 주식 밈 느낌
- 개미 캐릭터 중심의 게임 앱
- 직관적이고 재미있는 UX
- 너무 진지한 증권앱 느낌은 피함
- 도박, 리딩방, 자동매매 느낌은 절대 피함

## 디자인 원칙

1. **게임 앱답게**: 금융 앱이 아니라 소셜 게임 앱
2. **캐릭터 중심**: 개미 캐릭터가 앱의 주인공
3. **밈 활용**: 주식 밈 언어를 적절히 사용하되 도박 느낌 배제
4. **명확한 액션**: 버튼은 크고 명확하게, 다음 행동이 분명하게
5. **모바일 최적화**: 한 손 조작, 스크롤 최소화

## 컴포넌트

### 카드

- 둥근 모서리 (border-radius: 16px)
- 부드러운 그림자
- 패딩 16px
- 배경 흰색

### 버튼

- **Primary**: 주황 배경 + 흰색 텍스트, 둥근 모서리
- **Secondary**: 흰색 배경 + 주황 테두리
- **Danger**: 빨간 배경 (취소, 거절)
- 높이 48px 이상, 터치 영역 충분히
- 텍스트 Bold 16px

### 개미 캐릭터

MVP에서는 이모지 기반:
```
기본 개미:      🐜
승리 개미:      🐜👑
물림 개미:      🐜😢
떡상 개미:      🐜🔥
반등 기도:      🐜🙏
```

추후 커스텀 일러스트/SVG로 교체 가능하도록 컴포넌트화.

### 수익률 표시

```
+3.25%    → 빨간색, Bold, 위 화살표
-1.50%    → 파란색, Bold, 아래 화살표
 0.00%    → 회색, 가로줄
```

## 마이크로카피

### 홈 화면

- "개미 입장 완료"
- "오늘은 누가 물릴까?"
- "내 종목이 네 종목보다 잘 오를까?"

### 배틀 신청

- "민두개미가 배틀을 신청했어요!"
- "내일 종가까지 붙어볼까요?"
- "랜덤 배틀 기간을 돌려볼까요?"

### 종목 선택

- "종목 고르는 중... 신중한 척하는 중"
- "이미 상대 개미가 고른 종목이에요"
- "이 종목으로 배틀할까요?"

### 배틀 진행

- 앞서는 개미: "현재 우세!"
- 크게 앞서는 개미: "떡상 개미 출몰!"
- 뒤처지는 개미: "반등 기도중..."
- 크게 뒤처지는 개미: "물림 주의"
- 역전 발생: "역전! 개미판 뒤집혔다!"

### 배틀 결과

- "민두개미 승리!"
- "개미콩 +100 획득!"
- "반등 기도 실패... 그래도 참가 보상 +20"
- "오늘의 개미왕 등극!"
- "무승부! 둘 다 비슷한 개미"

### 빈 상태

- 친구 없음: "아직 친구가 없어요. 개미 친구를 찾아볼까요?"
- 배틀 없음: "진행 중인 배틀이 없어요. 배틀을 신청해볼까요?"
- 아이템 없음: "아직 아이템이 없어요. 배틀에서 개미콩을 모아보세요!"

## 애니메이션

### 개미 크기 변화

- `transform: scale()` 기반
- 300ms ease-in-out transition
- 수익률 변동에 따라 부드럽게 커지고 작아짐

### 화면 전환

- Stack navigator: 슬라이드 전환
- Tab navigator: 페이드 전환
- 모달: 바텀 시트 스타일

### 결과 화면

- 승리 시: 개미 바운스 애니메이션
- 개미콩 획득: 카운트 업 애니메이션
- 랭크 변동: 진행 바 애니메이션
